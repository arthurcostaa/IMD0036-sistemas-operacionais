Parte 1 - Manipulando processos com as funes bsicas

Para um processo criar outro no Linux, usa-se a funo fork(). Tal funo retorna um inteiro do tipo pid_t (tipo reservado para identificadores de processos), de acordo com as condies abaixo:

  • -1, caso haja erro na criao do novo processo;
  • 0, para o processo filho;
  • Um inteiro positivo representando o id do filho, para o processo pai.

Vejamos como isso acontece. Crie um arquivo C com o cdigo abaixo ou o execute no Colab do Google.

In[]:
%%writefile simpleFork.c
#include <sys/types.h>
#include <sys/wait.h>
#include <stdio.h>
#include <unistd.h>

int main(int argc, char **argv)
{
    pid_t pid=fork(); // Funcao usada para criar um novo processo

    if(pid<0){ // Erro na criacao do processo
        printf("Sou o processo %d e nao consegui criar um novo processo.\n",getpid());
        return 1;
    }else if (pid==0){ // Processo filho
        printf("Sou o processo filho, meu PID e %d.\n",getpid());
    }else{// Processo pai
        printf("Sou o processo pai do %d, meu PID e %d.\n",pid,getpid());
        wait(NULL); // Funcao que faz o processo pai esperar pelo final da execucao de um de seus filhos
    }

    return 0;
}

A funo wait() faz com que o pai espere pela execuo do filho antes de continuar rodando. Chame essa funo antes da linha printf("Sou o processo pai do %d, meu PID e %d.\n",pid,getpid()); e perceba que o processo filho passa a escrever no terminal antes do processo pai.

Alm disso, observe que o pai e o filho executam exatamente o mesmo cdigo. Isso acontece pois, quando usamos a funo fork(), o processo filho uma cpia do processo pai. Para alterarmos o programa que o filho est rodando, precisamos usar a funo exec() ou uma de suas derivadas.

Faremos isso a seguir. Primeiro, criaremos um novo codigo para usar no processo filho. Neste cdigo, faremos o filho imprimir uma frase no terminal.

Depois, executaremos uma versao modificada do codigo simpleFork, na qual passaremos esse novo programa para o processo filho.

In[]:
%%writefile forkWithExec.c
#include <sys/types.h>
#include <sys/wait.h>
#include <stdio.h>
#include <unistd.h>

int main(int argc, char **argv)
{
    pid_t pid=fork(); // Funcao usada para criar um novo processo

    if(pid<0){ // Erro na criacao do processo
        printf("Sou o processo %d e nao consegui criar um novo processo.\n",getpid());
        return 1;
    }else if (pid==0){ // Processo filho
        execlp("./newProgram","np",NULL); // Funcao derivada da exec()
        printf("Sou o processo filho, meu PID e %d.\n",getpid());
    }else{// Processo pai
        wait(NULL); // Funcao que faz o processo pai esperar pelo final da execucao de um de seus filhos
        printf("Sou o processo pai do %d, meu PID e %d.\n",pid,getpid());
    }

    return 0;
}

Parte 2 - Lidando com mltiplos forks

Vimos que o processo filho executa o mesmo programa do pai, a menos que a funo exec() seja chamada. O que acontece, ento, quando h mltiplos fork() e nenhum outro programa passado para os processos filhos?

Veremos essa resposta ao executar o cdigo a seguir.

In[]:
%%writefile threeForks.c
#include <sys/types.h>
#include <sys/wait.h>
#include <stdio.h>
#include <unistd.h>

int main(int argc, char **argv)
{
    fork(); // Funcao usada para criar um novo processo
    fork(); // E mais um
    fork(); // E mais um
    printf("Sou o processo %d.\n",getpid());
    return 0;
}

Perceba que temos 8 processos diferentes. Acompanhe o passo-a-passo do que aconteceu:

  • No incio, havia apenas o processo P0;
  • Com o primeiro fork(), P0 cria P1;
  • Com o segundo fork(), P0 e P1 criam P2 e P3, respectivamente;
  • Com o terceiro e ltimo fork(), P0, P1, P2 e P3 criam P4, P5, P6 e P7, respectivamente.

Outra dvida que pode surgir a seguinte: quando todos os processos manipulam uma mesma varivel, o que acontece?

In[]:
%%writefile twoProcessesOneVariable.c
#include <sys/types.h>
#include <sys/wait.h>
#include <stdio.h>
#include <unistd.h>
#include <stdlib.h>

int valor=100;
int main(int argc, char **argv)
{
    pid_t pid=fork(); // Funcao usada para criar um novo processo
    if(pid==0){// Processo filho
      valor+=100;
      exit(0); // Funcao que encerra o processo
    }else if(pid>0){ // Processo pai
      wait(NULL); // O pai vai esperar a conclusao do filho
    }
    printf("Valor da varivel e %d\n",valor);
    return 0;
}

Overwriting twoProcessesOneVariable.c
In[]:
!gcc ./twoProcessesOneVariable.c -o twoProcessesOneVariable
!./twoProcessesOneVariable

Valor da varivel e 100

Perceba que o nico processo que realiza a impresso de valor o processo pai, pois o filho j foi encerrado com a funo exit(). Observe tambm que a varivel permaneceu inalterada no processo pai, apesar do filho t-la incrementado. Isso aconteceu pois, quando o filho foi criado, ele copiou a varivel. Dessa maneira, ao fazer a operao valor+=100;, ele alterou apenas a sua prpria cpia e no a varivel presente no espao de endereamento do pai.
